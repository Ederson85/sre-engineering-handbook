# Linux Best Practices

**Module:** Linux  
**Category:** Best Practices

---

## Overview

These practices help improve reliability, reduce operational risk and standardize incident response.

---

## Best Practices

- Verify the target server before executing any command.
- Collect evidence before applying mitigation.
- Prefer graceful actions over forceful ones.
- Correlate infrastructure metrics with application behavior.
- Review logs before restarting services.
- Validate the impact of every action.
- Document every production incident.
- Keep runbooks up to date.
- Automate repetitive operational tasks.
- Continuously improve monitoring and alerting.
- Review capacity trends regularly.
- Learn from every incident through post-incident analysis.

---

## Daily SRE Checklist

- Confirm the alert.
- Validate the affected service.
- Identify the root cause.
- Apply the safest mitigation.
- Validate service recovery.
- Document findings.
- Share lessons learned.

---

# 💡 SRE Thinking

Operational excellence is built through consistency.

Reliable systems are the result of disciplined engineering, continuous improvement and well-defined operational practices.

Every incident is an opportunity to make the platform more resilient.
